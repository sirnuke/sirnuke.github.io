SteamBridge

Toolkit enabling Linux Steam users access to their Windows catalog, from within the native client.
C & C++, Python, x86 Visual Studio assembler, a bit of Ruby, BASH shell scripting

SteamBridge implements a pair of libraries that proxy the Steam API calls from an application running in Wine, to the native Steam client.

A Win32 DLL loads Linux code via a Winelib bridge library, and performs black magic assembler to translate between the two calling conventions.

Additionally, the toolkit comes with a set of Python scripts to configure and run applications using the SteamBridge environment.

And here's a fun piece of trivia: GCC x86 purely caller cleanup, expect when returning structs. The hidden pointer to the returned object is removed by the callee, I assume because REASONS.
